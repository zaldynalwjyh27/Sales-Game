# جسر - محاكي المبيعات (JISR - Sales Simulator)

## مشروع محاكاة المبيعات التفاعلي

### نظرة عامة
جسر (JISR) هو تطبيق ويب تفاعلي مصمم لتدريب مهارات المبيعات من خلال محاكاة لسيناريوهات بيع واقعية. يتيح التطبيق للمستخدمين التدريب على مهارات البيع، واستكشاف المشكلات، ومعالجة الاعتراضات، وإغلاق الصفقات في بيئة محاكاة.

### التكنولوجيا المستخدمة
- **Next.js 16.2.4** - إطار العمل الرئيسي
- **React 19.2.4** - مكتبة واجهة المستخدم
- **TypeScript** - لغة البرمجة
- **SQLite** - قاعدة البيانات (بواسطة Prisma)
- **Pusher** - للتواصل في الوقت الفعلي
- **OpenAI GPT-4o-mini** - الذكاء الاصطناعي للمحادثات
- **Tailwind CSS** - تنسيق الواجهة
- **Zustand** - إدارة الحالة

### قواعد البيانات (Prisma Schema)

#### النماذج:
- **Room**: غرف المحاكاة التي تحتوي على الجولة الحالية والسيناريو
  - [id] String @default(uuid()) - معرف الغرفة
  - status String @default("LOBBY") - الحالة (LOBBY, IN_PROGRESS, REVEALED)
  - currentRound Int @default(1) - الجولة الحالية
  - scenarioId Int? - معرف السيناريو
  - players Player[] - قائمة اللاعبين
  - messages Message[] - الرسائل في الغرفة
  - evaluations Evaluation[] - التقييمات

- **Player**: لاعبي المحاكاة
  - [id] String @default(uuid()) - معرف اللاعب
  - roomId String - معرف الغرفة
  - name String - اسم اللاعب
  - role String? - الدور (SELLER, CLIENT, EVALUATOR)
  - isHost Boolean @default(false) - هل هو المضيف؟
  - room Room - العلاقة مع الغرفة

- **Message**: الرسائل في المحادثة
  - [id] String @default(uuid()) - معرف الرسالة
  - roomId String - معرف الغرفة
  - senderId String? - معرف المرسل (null إذا كان ذكاء اصطناعي)
  - content String - محتوى الرسالة
  - isAi Boolean @default(false) - هل هي رسالة من الذكاء الاصطناعي؟
  - room Room - العلاقة مع الغرفة
  - sender Player? - العلاقة مع المرسل

- **Evaluation**: تقييم أداء اللاعبين
  - [id] String @default(uuid()) - معرف التقييم
  - roomId String - معرف الغرفة
  - evaluatorId String - معرف المقيم
  - targetId String - معرف الشخص المُقيم عليه
  - scores String - نتائج التقييم (JSON)
  - revealed Boolean @default(false) - هل تمت الإشارة إلى النتائج؟
  - room, evaluator, target - علاقات مع الكيانات الأخرى

### الوظائف الأساسية

#### 1. إنشاء الغرفة
- المضيف يدخل اسمه ويبدأ إنشاء غرفة
- يتم إنشاء غرفة بحالة "LOBBY"
- يتم إنشاء لاعب مضيف تلقائيًا

#### 2. الانضمام إلى الغرفة
- يتم إرسال رابط الغرفة إلى اللاعبين الآخرين
- عند الانضمام، يتم إنشاء لاعب جديد في الغرفة
- يتم إعلام جميع الأعضاء عبر Pusher

#### 3. توزيع الأدوار
- المضيف يضغط على "بدء التدريب وتوزيع الأدوار"
- يتم تعيين الأدوار التالية:
  - SELLER (البائع): مسؤول عن بيع المنتج
  - CLIENT (العميل): يمثل العميل المحتمل
  - EVALUATOR (المُقيم): يقيم أداء البائع

#### 4. التفاعل في المحاكاة
- إذا كان هناك عميل بشري، يتم التفاعل مباشرة
- إذا لم يكن هناك عميل بشري، يتفاعل الذكاء الاصطناعي كعميل
- يتحكم الذكاء الاصطناعي في الشخصية والسيناريو المحدد

#### 5. التقييم
- المُقيم يقيم أداء البائع بناءً على معايير محددة
- بعد انتهاء الجولة، يمكن لرئيس الغرفة كشف التقييمات
- يتم عرض النتائج بعد الكشف

### مكونات الذكاء الاصطناعي

#### الشخصيات (Personas)
- **مدير الموارد البشرية**: يركز على الامتثال، وتقليل العمل اليدوي، وتجربة الموظف
- **المدير المالي**: دقيق، يركز على الأرقام والعوائد المالية والمخاطر
- **مدير العمليات**: سريع، يركز على الكفاءة ومراقبة الدوام

#### السيناريوهات
- 12 سيناريو مختلف لمحاكاة مواقف بيع واقعية
- كل سيناريو له:
  - وصف الموقف
  - الألم المخفي (pain)
  - الاعتراض الأولي (objection)
  - سؤال المحفز (trigger question)
  - المنافس (إذا وُجد)

#### معايير التقييم
- يفتح المحادثة بسؤال مفتوح
- يكتشف 3 آلام في أقل من 15 دقيقة
- يعترف بالاعتراض قبل الرد
- يرد على الاعتراض بسؤال لا بجدال
- يخلق إلحاحاً طبيعياً
- يغلق بخطوة تالية محددة بتاريخ
- يرسل إيميل متابعة خلال ساعتين
- يستخدم الـ reference / proof point بشكل صحيح

### مكونات الواجهة

#### ChatInterface
- واجهة المحادثة بين الأدوار
- يعرض الرسائل في الوقت الفعلي
- يسمح بإرسال الرسائل الجديدة

#### RoleCardDisplay
- يعرض بطاقة تحتوي على معلومات الدور
- يعرض معلومات السيناريو حسب الدور
- يعرض معلومات الشخصية (إذا كان العميل)

#### HiddenEvaluatorForm
- نموذج للمُقيم لتقييم أداء البائع
- يحتوي على معايير التقييم الثمانية
- يرسل التقييم إلى قاعدة البيانات

#### ResultsRevealModal
- نافذة لعرض النتائج بعد كشفها
- يعرض تفاصيل التقييمات

### تدفق البيانات

1. المستخدم يفتح التطبيق ويختار إما إنشاء غرفة أو الانضمام إليها
2. عند بدء اللعبة، يتم توزيع الأدوار
3. يتم اختيار سيناريو عشوائي من قاعدة البيانات
4. تبدأ المحادثة بين الأدوار
5. إذا لم يكن هناك عميل بشري، يتفاعل الذكاء الاصطناعي كعميل
6. المُقيم يقيم أداء البائع
7. المضيف يمكنه كشف التقييمات لإنهاء الجولة
8. يمكن الانتقال إلى جولة جديدة مع سيناريو جديد

### ميزات Pusher (الاتصال في الوقت الفعلي)
- إشعارات عند انضمام اللاعبين
- بدء اللعبة وتوزيع الأدوار
- استقبال الرسائل الجديدة
- كشف التقييمات
- الانتقال إلى الجولة التالية
